# GPT-2 原理导读

**论文标题**：`Language Models are Unsupervised Multitask Learners`
**年份**：`2019`
**主题**：`大规模生成式预训练`
**定位**：让研究界第一次强烈感受到语言模型可以成为通用任务接口的关键论文。
**论文链接**：[Official PDF](https://cdn.openai.com/better-language-models/language_models_are_unsupervised_multitask_learners.pdf)

## 1. 代表公式 / 关键表达
- 核心训练目标仍是：`p(x) = Π_t p(x_t | x_{<t})`。
- 零样本任务可理解为：`p(y | instruction, examples, x)`，把任务本身放入上下文。

## 2. 这篇论文要解决什么问题
- 在 GPT-1 之后，一个核心疑问是：扩大模型和数据后，单纯语言模型是否会自然获得更强的任务迁移能力。
- 很多任务仍依赖显式微调，大家尚不清楚零样本能力会不会随着规模上升而出现。
- 作者希望验证：统一语言建模是否足以支持多任务泛化。

## 3. 核心思想
- 继续采用 decoder-only 因果语言建模，但显著扩大模型和语料规模。
- 强调零样本设置，让模型通过自然语言提示直接执行任务。
- 把“任务”视作上下文的一部分，而不必总是修改参数。

## 4. 方法原理拆解
- 模型结构并无革命性变化，真正变化在于规模、数据质量与训练坚持度。
- 通过给定任务描述或示例，让语言模型在上下文中完成翻译、摘要、问答等任务。
- 这篇的重要意义不在具体技巧，而在于展示规模本身如何改变模型行为。

## 5. 代码实现结构
- 从实现角度看，GPT-2 仍然是非常标准的 `decoder-only Transformer`，其代码骨架与 GPT-1 一脉相承。
- 结构上通常拆成 `Embedding 层`、`多层 Decoder Block`、`Final Norm`、`LM Head` 四部分。
- GPT-2 的关键不是新增复杂模块，而是把这种简单结构扩展到更大规模，并让推理接口开始服务零样本多任务场景。

### 5.1 Embedding 与输入组织
- 输入仍然是 `token embeddings + position embeddings`。
- 在源码里常见 `wte` 表示词嵌入，`wpe` 表示位置嵌入。
- 任务描述、示例和正式输入都被拼到同一串 token 中，因此从模型角度看，一切都是统一序列。

### 5.2 Decoder Block
- 每层核心仍是 `Masked Multi-Head Self-Attention + FFN`。
- GPT-2 通常把多个 block 放在 `ModuleList` 中顺序堆叠。
- 整个主干没有 encoder / decoder 分工，没有 cross-attention，也没有单独的句级 pooler。

### 5.3 因果注意力实现
- 因果 mask 依旧是 GPT-2 的关键基础设施。
- 由于模型规模更大，代码实现会更关注 attention 计算、cache 和矩阵布局的效率。
- 虽然 GPT-2 论文阶段 KV cache 还没有今天这么中心化讨论，但现代实现里几乎都会在推理时缓存历史 `K/V` 以避免重复计算。

### 5.4 Final Norm 与 LM Head
- GPT-2 常在所有 block 之后再接一个最终归一化层。
- `LM head` 将最后隐藏状态映射到词表 logits。
- 实现里也常共享 `token embedding` 与 `lm_head` 的权重，这是 GPT 系列非常常见的写法。

### 5.5 一个最小实现骨架

```python
class GPT2Model(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.wte = nn.Embedding(config.vocab_size, config.hidden_size)
        self.wpe = nn.Embedding(config.max_position, config.hidden_size)
        self.blocks = nn.ModuleList([GPTBlock(...) for _ in range(config.num_layers)])
        self.ln_f = nn.LayerNorm(config.hidden_size)
        self.lm_head = nn.Linear(config.hidden_size, config.vocab_size, bias=False)

    def forward(self, input_ids):
        pos = torch.arange(input_ids.size(1), device=input_ids.device).unsqueeze(0)
        x = self.wte(input_ids) + self.wpe(pos)
        causal_mask = build_causal_mask(input_ids.size(1), input_ids.device)
        for block in self.blocks:
            x = block(x, causal_mask)
        x = self.ln_f(x)
        return self.lm_head(x)
```

### 5.6 如果从 today 的工程视角理解 GPT-2
- 训练结构和推理结构本质相同，都是自回归预测下一个 token。
- 生成时会不断把新 token 接回输入，现代实现常配合 `past_key_values` 做增量推理。
- 从代码可维护性看，GPT-2 是最容易写出“最小可运行 LLM”的结构之一。

### 5.7 实现时最值得注意的点
- 零样本能力来自上下文组织，而不是额外任务头，因此 prompt 拼接方式本身就是“程序逻辑”的一部分。
- 长输入时，位置嵌入上限和上下文窗口大小会直接约束模型能力。
- 如果自己实现，最先要确保正确的是 `causal mask`、`shifted labels` 和 `weight tying`。

## 6. 训练或推理范式
- 核心策略是更大的数据和更大的模型，而非复杂任务头。
- 论文把零样本泛化作为主观察对象，推动大家重新审视微调是否总是必要。
- GPT-2 也让“预训练模型可能带来安全与滥用风险”首次大规模进入公众讨论。

## 7. 为什么它有效
- 更大模型拥有更强的统计压缩能力，能从无监督语料中吸收更多隐含规律。
- 自然语言提示本身携带任务信息，因此无需显式参数更新也能部分迁移。
- 统一目标减少了多任务之间的接口碎片化。

## 8. 局限与争议
- 零样本能力仍不稳定，提示形式对结果影响较大。
- 许多任务效果还不能替代专门微调模型。
- 缺少系统化的 few-shot 与 in-context learning 理论解释。

## 9. 对后续研究的影响
- 为 GPT-3 的 few-shot 时代打下实验与叙事基础。
- 把 prompt 作为任务接口的思想提前暴露出来。
- 加速了社区对“大模型是否会自然涌现更多能力”的探索。

## 10. 你应该怎样读这篇
- 读这篇时不要只盯着分数，要注意作者对零样本多任务能力的叙述方式。
- 它是从 GPT-1 到 GPT-3 的过渡桥梁，历史意义大于结构创新。
- 如果你关心 prompt engineering 的源头，这篇值得细看。
