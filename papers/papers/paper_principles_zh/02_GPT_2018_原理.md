# GPT 原理导读

**论文标题**：`Improving Language Understanding by Generative Pre-Training`
**年份**：`2018`
**主题**：`预训练 / 解码器`
**定位**：GPT 系列的起点，首次系统化展示生成式预训练如何迁移到多个下游任务。
**论文链接**：[Official PDF](https://cdn.openai.com/research-covers/language-unsupervised/language_understanding_paper.pdf)

## 1. 代表公式 / 关键表达
- 自回归语言模型目标：`p(x) = Π_t p(x_t | x_{<t})`。
- 迁移时可理解为在预训练目标之上，对下游任务条件分布 `p(y | x)` 做轻量适配。

## 2. 这篇论文要解决什么问题
- 传统监督学习高度依赖标注数据，而高质量标注成本很高，迁移性也有限。
- 研究者希望找到一种更通用的预训练方式，让模型先从无标注语料中学语言，再迁移到具体任务。
- 核心问题是：只用单向语言建模，能否学到足够强的任务通用知识。

## 3. 核心思想
- 采用标准的从左到右语言模型目标进行预训练，最大化下一个 token 的概率。
- 再把预训练参数迁移到下游任务，通过最少结构改动进行微调。
- 保留统一的文本接口，让不同任务都尽量映射到条件生成或分类形式。

## 4. 方法原理拆解
- 模型采用 decoder-only Transformer，并使用因果 mask 保证每个位置只能看到左侧上下文。
- 这种单向建模天然适配文本生成，训练目标也与推理方式保持一致。
- 论文通过任务适配输入模板，让同一个主干可以做分类、蕴含、相似度等多类任务。

## 5. 代码实现结构
- 如果从代码结构看，GPT-1 通常可以拆成 `Token/Position Embedding`、`Decoder Block 堆叠`、`LM Head`、`任务适配头` 四部分。
- 与 BERT 最大不同在于：GPT 只有 decoder 栈，没有 encoder，也没有双向可见性。
- 工程实现主干通常可抽象为：`GPTEmbeddings -> DecoderBlocks -> Final Layer -> LM Head`。

### 5.1 Embedding 层
- `token embeddings`：把词 id 映射为连续向量。
- `position embeddings`：为每个位置加入顺序信息。
- GPT-1 一般不需要 BERT 那种 `token_type embeddings`，因为核心任务是单序列自回归建模。
- 输入嵌入一般是词向量与位置向量相加后进入 decoder block。

### 5.2 Decoder Block 主干
- GPT 的核心单元是 `Masked Multi-Head Self-Attention + FFN`。
- 每一层都通过因果 mask 保证位置 `t` 只能看到 `0..t` 的内容，不能偷看未来 token。
- 典型结构仍是 Transformer 风格：
  `x -> MaskedSelfAttention -> Add&Norm -> FFN -> Add&Norm`

### 5.3 因果 Self-Attention
- 与 BERT 的双向 self-attention 相比，GPT 这里最关键的是下三角 mask。
- 实现时会先算 `QK^T / sqrt(d)`，再把未来位置对应分数设为极小值，然后做 softmax。
- 这个 mask 让训练目标和推理目标保持一致，因为生成时本来就只能基于左侧上下文。

### 5.4 前馈网络
- 每层 attention 后都接一个两层 MLP。
- 典型实现是 `hidden_size -> intermediate_size -> hidden_size`。
- 激活函数与层归一化具体形式可能随实现不同而略有变化，但整体模块划分基本一致。

### 5.5 LM Head 与任务适配
- `LM head` 负责把最后一层隐藏状态投影回词表维度，得到下一个 token 的 logits。
- 训练时目标是让位置 `t` 预测 `t+1` 的 token。
- GPT-1 论文里很多下游任务并不是完全另起一套模型，而是尽量通过输入格式改写，复用同一个生成式主干。

### 5.6 一个最小实现骨架

```python
class GPTBlock(nn.Module):
    def __init__(self, hidden_size, num_heads, intermediate_size):
        super().__init__()
        self.attn = CausalSelfAttention(hidden_size, num_heads)
        self.attn_norm = nn.LayerNorm(hidden_size)
        self.ffn = nn.Sequential(
            nn.Linear(hidden_size, intermediate_size),
            nn.GELU(),
            nn.Linear(intermediate_size, hidden_size),
        )
        self.ffn_norm = nn.LayerNorm(hidden_size)

    def forward(self, x, causal_mask):
        x = self.attn_norm(x + self.attn(x, causal_mask))
        x = self.ffn_norm(x + self.ffn(x))
        return x


class GPTModel(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.token_embeddings = nn.Embedding(config.vocab_size, config.hidden_size)
        self.position_embeddings = nn.Embedding(config.max_position, config.hidden_size)
        self.blocks = nn.ModuleList([GPTBlock(...) for _ in range(config.num_layers)])
        self.lm_head = nn.Linear(config.hidden_size, config.vocab_size, bias=False)

    def forward(self, input_ids):
        pos = torch.arange(input_ids.size(1), device=input_ids.device).unsqueeze(0)
        x = self.token_embeddings(input_ids) + self.position_embeddings(pos)
        causal_mask = build_causal_mask(input_ids.size(1), input_ids.device)
        for block in self.blocks:
            x = block(x, causal_mask)
        return self.lm_head(x)
```

### 5.7 实现时最值得注意的点
- 因果 mask 是 GPT 系列最核心的结构差异，很多 bug 都出在 mask 维度和广播方式上。
- `lm_head` 常与词嵌入矩阵做权重共享，以减少参数并保持输入输出词表映射一致。
- 训练时 label 一般会右移一位，因此实现里常见 `shift_logits` 和 `shift_labels`。
- GPT-1 虽然早期规模不大，但代码结构已经奠定了后续 GPT-2 / GPT-3 的基本实现形态。

## 6. 训练或推理范式
- 先做大规模无监督预训练，再在有标签数据上轻量微调。
- 这篇论文在规模上并不夸张，但方法论非常关键，后续 GPT-2 与 GPT-3 基本沿着这条路扩展。
- 它强调预训练目标简单但通用，尽量避免任务特定设计。

## 7. 为什么它有效
- 语言建模本身迫使模型学习语法、语义和世界知识的压缩表达。
- 解码器结构对开放式生成友好，因此具备更自然的任务统一接口。
- 训练目标与生成式推理过程一致，避免了部分预训练与推理分布错位问题。

## 8. 局限与争议
- 单向上下文在纯理解任务上不一定像双向建模那样高效。
- 当时模型规模还不够大，许多能力没有完全显现出来。
- 任务适配仍需要一定手工设计，尚未发展到后来的 prompt 统一接口。

## 9. 对后续研究的影响
- 明确建立了 GPT 路线，为后来的 GPT-2、GPT-3、ChatGPT 铺平道路。
- 证明了生成式预训练不是只能做续写，也可以迁移到理解任务。
- 把 decoder-only Transformer 推向基础模型主线。

## 10. 你应该怎样读这篇
- 重点看作者为什么相信“语言模型就是通用预训练目标”。
- 这篇最好和 BERT、T5 一起读，帮助建立三条主流架构的地图。
- 如果你想理解今天为什么很多模型仍坚持 decoder-only，这篇是逻辑起点。
