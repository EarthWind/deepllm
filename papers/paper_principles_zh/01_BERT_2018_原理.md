# BERT 原理导读

**论文标题**：`BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding`
**作者**：Jacob Devlin 等，Google AI，2018年10月
**年份**：`2018`
**主题**：`预训练 / 编码器`
**定位**：Transformer 在 NLP 预训练时代真正大规模进入主流的标志性论文。
**论文链接**：[arXiv](https://arxiv.org/abs/1810.04805)

BERT 的核心贡献可以从三个层面理解：它解决的问题、它的技术创新，以及它带来的范式转变。先从最关键的直觉开始——双向性。可视化工具暂时无响应，我直接用文字详细介绍 BERT。

**它解决了什么根本问题**
在 BERT 之前，NLP 预训练模型面临一个两难困境：
语言模型天然是"单向"的——预测下一个词时，你只能看到左边的词（GPT）；或者把左向和右向分开训练再拼接（ELMo），但两个方向的表示从未在同一层内真正交互，是"浅层双向"。

理解语言却需要双向上下文。"他去银行取了钱"和"他去河岸钓了鱼"，"银行/岸"这两个字的意思完全靠左右上下文共同决定。

BERT 的核心洞察：**用 Transformer 的自注意力机制，让每个 token 在同一层内同时关注句子里所有其他 token**——这才是真正的深度双向融合。

**两大预训练任务（最核心的创新）**

**任务 1：Masked Language Model（MLM）**
随机遮盖输入中 15% 的 token，让模型预测被遮盖的词。例如：
```
输入：  我 今天 去 [MASK] 吃 了 饭
目标：  预测 [MASK] = "食堂"
```

这个设计巧妙地绕开了单向限制——因为是预测"中间被盖住的词"，模型必须同时看左右两侧的上下文才能做出预测，从而被迫学习双向表示。
具体操作：被选中的 15% 中，80% 替换为 `[MASK]`，10% 替换为随机词，10% 保持不变。这样做的原因是：防止模型在 fine-tune 阶段遇到没有 `[MASK]` 的真实句子时产生分布偏差。

**任务 2：Next Sentence Prediction（NSP）**
给模型两个句子 A 和 B，让它判断 B 是不是 A 的真实下一句（50%是，50%随机）：

```
输入：[CLS] 今天天气很好 [SEP] 我们去公园吧 [SEP]  → IsNext
输入：[CLS] 今天天气很好 [SEP] 量子力学很难 [SEP]  → NotNext
```

这让模型学习句子级别的关系，对问答、自然语言推理等需要理解跨句关系的任务很有帮助（不过后来 RoBERTa 证明 NSP 收益有限，争议较大）。

BERT 引入了几个有特殊含义的标记：

- `[CLS]`（Classification）放在句子开头，其对应的输出向量代表整个句子的表示，用于分类任务
- `[SEP]`（Separator）分隔两个句子
- `[MASK]` 表示被遮盖的词

Position Embedding 和 Segment Embedding（区分句子A/B）加上 Token Embedding，三者相加构成最终输入。


## 1. 代表公式 / 关键表达
- MLM 目标可写成：`L_MLM = - Σ_{i in M} log p(x_i | x_{\M})`，只在被遮盖位置上计算损失。
- NSP 可以理解为句对分类：`p(is_next | [CLS])`，用于学习句间关系。

## 2. 这篇论文要解决什么问题
- 在 BERT 之前，很多模型虽然能在单个任务上表现不错，但缺少统一、可迁移的大规模预训练方案。
- 早期语言模型多是单向的，擅长预测下一个词，但对句子级理解、实体关系、语义匹配等任务不够自然。
- 作者想证明：如果让 Transformer 编码器先在海量文本上学到深层双向表示，再针对下游任务微调，就能显著提升通用 NLP 能力。

## 3. 核心思想
- 使用 Masked Language Modeling，让模型在看到左右上下文的条件下恢复被遮住的 token。
- 引入 Next Sentence Prediction，尝试让模型学习句间关系与篇章级连贯性。
- 采用统一预训练加微调范式，下游任务只需要在顶层接一个很小的任务头。

## 4. 方法原理拆解
- 模型本体是多层 Transformer encoder，没有 decoder，因此天然更适合理解而非生成。
- MLM 通过随机替换部分 token，让模型同时利用左侧和右侧上下文，得到真正的双向表示。
- 输入由 token embedding、segment embedding 和 position embedding 组成，使模型能区分句子 A/B 与位置。
- 预训练完成后，分类任务通常取 [CLS] 表示，序列标注则取每个 token 的隐藏状态。

## 5. 代码实现结构
- 如果按工程实现拆分，BERT 通常可以分成 `Embedding 层`、`Encoder 主干`、`Pooler`、`预训练头`、`下游任务头` 五部分。
- 真正的核心主干是 `BertEmbeddings -> BertEncoder -> BertPooler`，而 `MLM / NSP` 只是预训练阶段挂在顶部的头。
- 绝大多数实现里，`BERT base` 与 `BERT large` 的区别主要体现在层数、隐藏维度、attention heads 数量，而不是代码结构变化。

### 5.1 Embedding 层
- `token embeddings`：把词表 id 映射成向量。
- `position embeddings`：给每个位置一个绝对位置向量。
- `token_type embeddings`：区分句子 A / 句子 B，用于句对任务。
- 三者相加后，通常接 `LayerNorm + Dropout`，这一步构成编码器输入。

### 5.2 Encoder 主干
- Encoder 由多层完全相同的 `Transformer Encoder Block` 堆叠而成。
- 每层通常包含两个子模块：`Multi-Head Self-Attention` 和 `Feed-Forward Network`。
- 每个子模块外面都带残差连接和归一化，因此典型结构可理解为：
  `x -> SelfAttention -> Add&Norm -> FFN -> Add&Norm`
- 由于 BERT 是 encoder-only，因此 self-attention 不使用因果 mask，而是允许当前位置看到双向上下文。

### 5.3 Attention 子模块
- 输入 `hidden_states` 先线性投影成 `Q / K / V`。
- 然后按 head 维度切分，计算 `softmax(QK^T / sqrt(d_k))V`。
- 多头结果拼接后再过一个输出线性层，回到原隐藏维度。
- 代码实现上最容易出错的地方是 `mask` 的广播维度、`reshape / view / transpose` 顺序，以及 attention score 的数值稳定性。

### 5.4 Feed-Forward 子模块
- 每层 attention 之后通常接两层 MLP：`hidden_size -> intermediate_size -> hidden_size`。
- 原论文里常用 `GELU` 作为激活函数，这是 BERT 时代一个很典型的实现细节。
- `intermediate_size` 往往比 `hidden_size` 大很多，例如 base 版常见是 4 倍扩张。

### 5.5 Pooler 与句级表示
- BERT 常把最后一层的 `[CLS]` token 表示作为句级摘要表示。
- 实现里通常再经过一个线性层和 `tanh`，得到 `pooled_output`。
- 这个 `pooled_output` 最初是为了 NSP 和分类任务方便，不代表它一定是所有任务的最优句向量。

### 5.6 预训练头
- `MLM head`：接在每个 token 的隐藏状态上，预测被 mask 掉位置的词。
- `NSP head`：接在 `[CLS]` 对应的 `pooled_output` 上，判断句子 B 是否真的是句子 A 的下一句。
- 代码层面上，预训练模型常写成：
  `BertModel + BertPreTrainingHeads`
- 其中 `BertModel` 输出主干表示，`BertPreTrainingHeads` 只负责把表示映射到具体训练目标。

### 5.7 下游任务头
- 文本分类：在 `[CLS]` 或 pooled_output 上接线性分类器。
- 序列标注：对每个 token 的最后层隐藏状态接分类层。
- 问答：对每个 token 预测 `start logits` 和 `end logits`。
- 这也是为什么 BERT 很适合作为“统一底座”，因为主干不动，只需要替换顶部小头部。

### 5.8 一个最小实现骨架

```python
class BertEmbeddings(nn.Module):
    def __init__(self, vocab_size, hidden_size, max_pos, type_vocab_size):
        super().__init__()
        self.word_embeddings = nn.Embedding(vocab_size, hidden_size)
        self.position_embeddings = nn.Embedding(max_pos, hidden_size)
        self.token_type_embeddings = nn.Embedding(type_vocab_size, hidden_size)
        self.norm = nn.LayerNorm(hidden_size)
        self.dropout = nn.Dropout(0.1)

    def forward(self, input_ids, token_type_ids):
        seq_len = input_ids.size(1)
        position_ids = torch.arange(seq_len, device=input_ids.device).unsqueeze(0)
        x = (
            self.word_embeddings(input_ids)
            + self.position_embeddings(position_ids)
            + self.token_type_embeddings(token_type_ids)
        )
        return self.dropout(self.norm(x))


class BertLayer(nn.Module):
    def __init__(self, hidden_size, num_heads, intermediate_size):
        super().__init__()
        self.attn = MultiHeadSelfAttention(hidden_size, num_heads)
        self.attn_norm = nn.LayerNorm(hidden_size)
        self.ffn = nn.Sequential(
            nn.Linear(hidden_size, intermediate_size),
            nn.GELU(),
            nn.Linear(intermediate_size, hidden_size),
        )
        self.ffn_norm = nn.LayerNorm(hidden_size)

    def forward(self, x, attention_mask):
        x = self.attn_norm(x + self.attn(x, attention_mask))
        x = self.ffn_norm(x + self.ffn(x))
        return x


class BertModel(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.embeddings = BertEmbeddings(...)
        self.layers = nn.ModuleList([BertLayer(...) for _ in range(config.num_layers)])
        self.pooler = nn.Linear(config.hidden_size, config.hidden_size)

    def forward(self, input_ids, token_type_ids, attention_mask):
        x = self.embeddings(input_ids, token_type_ids)
        for layer in self.layers:
            x = layer(x, attention_mask)
        cls = torch.tanh(self.pooler(x[:, 0]))
        return x, cls
```

### 5.9 从源码角度最值得注意的点
- `attention_mask` 通常不是简单的 0/1 乘法，而是先扩展维度，再把 padding 位变成极小值加到 attention score 上。
- `MLM` 训练时只对被选中的 mask 位置计算 loss，因此实现上常要配合 `ignore_index`。
- 很多框架会把 `embedding weights` 与 `MLM decoder weights` 绑定，以减少参数并保持词表映射一致。
- 如果你对照 Hugging Face 的实现，通常会看到 `BertModel`、`BertForPreTraining`、`BertForSequenceClassification` 这样的分层类设计。

## 6. 训练或推理范式
- 训练阶段重点在海量通用语料上做表示学习，而不是针对单个 benchmark 做特化。
- 微调阶段几乎不改动主干结构，这种“一个底座适配多任务”的做法对后来的大模型影响极大。
- BERT 的成功也说明，预训练语料和优化策略本身就能产生强迁移效果。

这是 BERT 最深远的影响，超过了技术细节本身。

BERT 将 NLP 的工作流程定义为两阶段：

**阶段一：大规模无监督预训练**（在 BooksCorpus + Wikipedia 上，约 33 亿词）
训练一个通用的语言表示模型，不针对任何具体任务。

**阶段二：下游任务微调（Fine-tuning）**
只需在预训练模型上加一个简单的输出层，用少量标注数据训练几个 epoch，即可在各种任务上达到甚至超越专门设计的模型。

在 2018 年底，BERT 一次性刷新了 11 项 NLP 基准的 SOTA，包括 GLUE、SQuAD、MultiNLI 等。这震惊了整个学界——过去每个任务需要精心设计的特定架构，现在一个通用模型加简单微调就能全部搞定。

## 7. 为什么它有效
- 双向上下文比单向上下文更适合理解任务，因为模型可以同时参考前后语义。
- 统一预训练底座显著减少了下游任务从零训练的成本。
- Transformer 的全局注意力让模型更容易学习长距离依赖和句法语义关系。

## 8. 局限与争议
- MLM 训练目标和真实推理时的输入分布不完全一致，因为推理时通常不会出现 [MASK]。
- NSP 后来被证明并不是决定性设计，许多后续工作将其替换或删除。
- BERT 的 encoder-only 架构不适合开放式长文本生成，因此不直接通向聊天式 LLM 主线。

## 9. 对后续研究的影响
- 奠定了现代预训练加微调范式，并推动 NLP 从任务专用模型走向基础模型。
- 帮助研究社区明确区分 encoder-only、decoder-only、encoder-decoder 三大路线。
- 即使今天主流聊天模型不直接用 BERT 架构，很多表示学习、检索、分类任务仍深受其影响。

## 11. 模型规模

BERT 发布了两个版本：

- `BERT-base`：12 层 Transformer，768 维，12 个注意力头，1.1 亿参数
- `BERT-large`：24 层，1024 维，16 个注意力头，3.4 亿参数

在 4 个 TPU Pod（64块 TPU v3）上训练了 4 天，这在当时是相当大的算力投入。

## 12. 历史地位与局限

BERT 的意义在于它把"迁移学习"从计算机视觉（ResNet/ImageNet 预训练）带入了 NLP，证明了"先大规模预训练，再小样本微调"在语言理解任务上完全可行，并且效果惊人。

它的局限同样清晰：

- `[MASK]` 在预训练和推断阶段存在分布不一致（推断时没有 mask）
- 对**生成任务**不擅长，因为它是 Encoder-only 架构，预测的是填空而非自回归生成
- NSP 任务的实际收益被后来研究质疑
- 不能做 autoregressive 的文本生成（GPT 系列更适合这个）

这些局限直接催生了后来的 RoBERTa（优化训练）、XLNet（排列语言模型）、T5（统一 text-to-text）、GPT-2/3（坚持单向但做大）等一系列工作。
