# T5 原理导读

**论文标题**：`Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer`
**年份**：`2019/2020`
**主题**：`统一任务接口 / 编码器-解码器`
**定位**：把几乎所有 NLP 任务都统一成 text-to-text 的经典论文。
**论文链接**：[arXiv](https://arxiv.org/abs/1910.10683)

## 1. 代表公式 / 关键表达
- 统一目标可写成：`p(y_text | x_text)`，所有任务都被转成文本到文本映射。
- 预训练的 span corruption 本质是让模型恢复被遮盖片段：`p(span_targets | corrupted_input)`。

## 2. 这篇论文要解决什么问题
- NLP 任务接口长期非常碎片化，不同任务常常有不同模型头和不同数据格式。
- 这让迁移学习、比较实验和系统复用变得复杂。
- 作者希望提出一种统一视角：所有任务都写成文本输入到文本输出。

## 3. 核心思想
- 使用 encoder-decoder Transformer，天然适合处理条件生成。
- 把分类、翻译、摘要、问答都转成 text-to-text 形式。
- 通过 span corruption 等预训练目标提升模型对文本重建和条件生成的能力。

## 4. 方法原理拆解
- 编码器负责理解输入文本，解码器负责按目标格式生成输出。
- text-to-text 统一接口让模型可以用一套方法处理离散标签、自然语言答案和长文本生成。
- 预训练目标通常遮盖连续片段而不是单个 token，使模型更擅长恢复更高层语义结构。

## 5. 代码实现结构
- T5 的代码结构与 BERT、GPT 最大差异在于它是 `encoder-decoder`，因此主干天然分成两部分。
- 从工程实现上，通常拆成 `Shared Embedding`、`Encoder Stack`、`Decoder Stack`、`LM Head` 四块。
- 如果你看源码，最关键的不是单层 block 本身，而是 `decoder` 中多出来的 `cross-attention`。

### 5.1 Shared Embedding
- T5 常把输入端和输出端词表嵌入共享，减少参数量并统一词空间。
- 编码器输入和解码器输入都来自同一套 token embedding。
- 这种共享设计也方便 text-to-text 统一接口，因为输入输出本来都被视作文本序列。

### 5.2 Encoder Stack
- Encoder 结构和 BERT 类似，由若干层 `Self-Attention + FFN` 堆叠而成。
- 不同点在于 T5 常使用相对位置偏置，而非传统绝对位置 embedding 写法。
- Encoder 负责把输入文本编码成上下文表示，供 decoder 条件生成使用。

### 5.3 Decoder Stack
- Decoder 每层通常有三个子模块：`Masked Self-Attention`、`Cross-Attention`、`FFN`。
- `Masked Self-Attention` 负责让 decoder 只看到已生成前缀。
- `Cross-Attention` 负责让 decoder 读取 encoder 输出，因此它是 T5 条件生成能力的关键。

### 5.4 Encoder-Decoder 的前向路径
- 编码阶段：`encoder_hidden = Encoder(input_ids, attention_mask)`
- 解码阶段：`decoder_hidden = Decoder(decoder_input_ids, encoder_hidden, encoder_mask)`
- 最后通过 `lm_head` 把 decoder 隐藏状态映射到词表分布。
- 从代码上看，T5 的核心就是“编码输入文本，再条件生成输出文本”。

### 5.5 Span Corruption 的实现视角
- 预训练时不是简单 mask 单个 token，而是遮盖连续 span。
- 目标序列通常只包含被遮盖片段及其对应的特殊 sentinel token。
- 因此数据预处理在 T5 里非常关键，很多复杂度不在模型层，而在样本构造逻辑。

### 5.6 一个最小实现骨架

```python
class T5Model(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.shared = nn.Embedding(config.vocab_size, config.hidden_size)
        self.encoder = T5Encoder(self.shared, config)
        self.decoder = T5Decoder(self.shared, config)
        self.lm_head = nn.Linear(config.hidden_size, config.vocab_size, bias=False)

    def forward(self, input_ids, attention_mask, decoder_input_ids, decoder_attention_mask):
        encoder_hidden = self.encoder(input_ids, attention_mask)
        decoder_hidden = self.decoder(
            decoder_input_ids,
            decoder_attention_mask,
            encoder_hidden,
            attention_mask,
        )
        return self.lm_head(decoder_hidden)
```

### 5.7 下游任务头为什么很少单独强调
- 因为 T5 的很多任务直接通过文本生成完成，不一定需要额外分类头。
- 分类任务可以直接生成标签词，例如 `"positive"` 或 `"negative"`。
- 这也是 T5 统一接口思想在代码层面的体现：大量任务共享同一个输出头。

### 5.8 实现时最值得注意的点
- 自己实现 T5 时，最容易混淆的是 `encoder self-attention`、`decoder self-attention` 和 `cross-attention` 三种 mask 逻辑。
- span corruption 依赖专门的数据构造，不是模型 forward 里自然就有。
- 如果对照 Hugging Face，通常会看到 `T5Model`、`T5ForConditionalGeneration` 这样的分层设计，其中后者负责训练目标与生成接口。

## 6. 训练或推理范式
- 训练流程强调任务统一、数据配方、模型规模和预训练目标的系统性比较。
- T5 对后续 instruction tuning 的最大启发在于：任务应该尽量用自然语言描述。
- 其数据与训练工程同样重要，不只是架构选择。

## 7. 为什么它有效
- 统一接口降低了任务切换成本，也让模型更容易学习自然语言形式的任务说明。
- encoder-decoder 架构在条件生成上比纯 decoder 更直接、控制更清晰。
- span corruption 帮助模型学习局部与全局文本重建能力。

## 8. 局限与争议
- encoder-decoder 架构在超大规模生成式服务中的推理成本可能高于 decoder-only。
- 后来的聊天模型主线更多偏向 decoder-only，因此 T5 不再是主流商业接口形态。
- 尽管统一格式强大，但很多工程系统最终仍会针对特定任务进一步优化。

## 9. 对后续研究的影响
- 深刻影响了 instruction tuning、prompt 化任务接口和统一基础模型设计。
- 使“自然语言就是编程接口”这一思想更系统化。
- 成为理解 encoder-decoder 在大模型时代价值的代表论文。

## 10. 你应该怎样读这篇
- 读 T5 时，重点是理解“任务统一”而不是只看模型大小。
- 如果你经常处理结构化输出、摘要、翻译，这篇会给你很强的方法论。
- 把它和 GPT-3 对照看，能看出为什么后来商业系统多用 decoder-only，但研究思想仍大量继承 T5。
